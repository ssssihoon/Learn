### 머신러닝

- 인간의 학습 능력과 같은 기능을 컴퓨터에서 실현하고자 하는 기술과 기법을 의미

- 전통적인 프로그래밍
    - 모든 규칙들이 사전에 정의
    - 사전에 정의된 규칙과 데이터가 결합하여 출력을 생성
    - **단점 - 규칙이 추가될 경우 유지관리가 어려움**
- 머신러닝
    - 데이터를 학습하여 스스로 규칙을 생성
    - **장점 - 새로운 데이터에 알맞은 규칙을 추가로 생성할 필요 없음**
    - **전통적인 프로그래밍 대비 계산 효율이 좋음**

    1. 새로운 데이터를 이용하여 학습한 모델 성능 검증
    2. 교차 검증

### 프로그래밍 언어

- 컴퓨터가 수행할 명령을 전달하기 위해 작성할 때 사용하는 기호 체계를 의미
- 컴퓨터는 2진수로 조합된 기계어만을 이해하기 때문에 사람이 구사하는 자연어와의 의사소통을 이루기 위해 프로그래밍 언어가 필요

### 저급 언어와 고급 언어

저급언어

- 컴퓨터가 쉽게 이해하도록 프로그램을 작성할 때 사용하는 언어로 사람이 이해하기에는 다소 어려움
    - 기계어, 어셈블리어

고급언어

- 사람이 쉽게 이해하고 프로그램을 수월하게 작성할 수 있도록 도움을 주는 언어
    - 포트란, 코볼, 베이직, 파이썬, 파스칼, C, C++, 자바

### 프로그램 실행 절차

1. 자연어
2. 컴파일러 (프로그래밍 언어 to 기계어)
3. 프로그램 실행

### 컴파일

- 프로그래밍 언어를 통해 자연어를 기계어로 변환하는 과정
- 소스 코드에 대해 컴파일을 수행하게 되면 컴퓨터가 명령을 수행할 수 있는 실행파일이 생성됨

### 컴파일러

- 고급 언어로 작성된 원시 프로그램을 입력받아 컴퓨터가 이해할 수 있도록 기계어로 언어를 번역해 주는 프로그램 또는 소프트웨어

## 요구 사항 분석 명세서

- 개발 초기에 사용자의 요구사항을 추출하여 정리한 문서
- **시스템의 기능이 무엇인지에만 초점**을 두어 정리하고 어떻게 구현할지는 기술하지 않음
- 사용자 요구사항을 만족시키기 위해서는 **적시성, 유연성, 통합**에 대한 특성을 만족시켜야 함.

### 적시성

- 사용자가 원하는 시기에 맞출 수 있는 시간성을 의미
- 프로젝트는 시대적 흐름과 산업의 변화에 대해 환경의 적시성을 최대한 반영해야 함.

### 유연성

- 급변하는 환경에도 잘 적응할 수 있는 시스템
- 요구사항이 무엇인지에 대해 정확하고 구체적으로 분석하여 유연성을 확보해 두는 것은 미래를 위한 현명한 선택임.

### 통합

- 기존의 시스템과도 쉽게 통합할 수 있어야함을 의미

## 문제 정의

- 무엇을 개발할 것인지에 대해 명확히 정의하고 개발하고자 하는 범위를 결정
- 문제를 정의하려면 개발하고자 하는 영역의 배경 지식 또한 필요함

## 타당성 분석

- 개발하고자 하는 프로젝트의 신규 시스템 실현 방안을 모색하면서 투자 효율성과 시장 규모 → 경제적 타당성
- 명확한 의사소통으로 정의된 문제에 대해 사용자가 원하는 수준으로 개발하기 위한 집약된 기술력은 확보되었는지 → 기술적 타당성
- 개발과정에서 사용하는 프로그램이나 도구에 대한 라이선스 등의 법적인 문제는 없는지 → 법적 타당성

### 경제적 타당성

- 얼마만큼의 비용을 들여 개발한 시스템이 시작에서 얼마나 많은 이윤을 남겨줄 것인지
- 투자한 금액을 회수하고 영업이익이 발생하는 시점은 언제부터인지
- 투자 효율성을 충분히 검토하기 위해 경제적 타당성을 고려

### 기술적 타당성

- 사용자가 요구하는 프로젝트를 개발하기 위해 현재 기술로 기능을 충분히 구현할 수 있는지
- 프로젝트를 개발하기 위한 하드웨어 성능은 문제가 없는지, 개발자의 집약된 기술력에는 문제가 없는지
- 기술적 타당성을 검증하려면 유사 프로젝트의 사례를 선행 연구하여 분석

### 법적 타당성

- 프로그램이나 도구에 대한 라이선스 등의 법적인 문제는 없는지

### 작업 분할 구조도 WBS

- 프로젝트 목표를 달성하기 위해 수행하는 과정에서 필요한 활동과 업무를 세분화하는 작업을 의미
- WBS는 프로젝트의 구성요소들을 계층 구조로 분류하여 프로젝트의 전체 범위를 정의하고 프로젝트 작업을 관리하기 쉽도록 작게 세분화된 Task 목록을 리스트로 제시
    - 작업 패키지 : 계층 구조에서 최하위에 있는 항목을 의미
    - 태스크 : 작게 나누어진 작업 단위를 의미

### WBS의 목적과 활용도

- 사용자와 개발자 간의 의사소통 도구로 사용함
- 프로젝트 업무 내역을 가시화 할 수 있으므로 관리가 용이함
- 프로젝트 팀원의 책임과 역할이 분명하게 구분되므로 업무 분담의 효율화를 가할 수 있음
- 필요 인력과 일정 계획을 세우는 데 기초 근거로 활용
- 개발비 산정 시 산출근거로 활용
- 성과 측정 및 조정 시 기준선으로 활용

### 추상화

- 문제와 관련된 핵심 정보에만 초점을 둠

### 일반화

## 알고리즘

- 주어진 문제를 논리적으로 해결하기 위해 필요한 절차와 방법 및 명령어들을 모아 놓은 것을 의미

### 알고리즘을 사용하는 이유

- 컴퓨터는 사람이 하는 대화를 그대로 받아들여 명령을 수행하지 못하기 때문에 알고리즘이 필요함

- 출력

- 효율성

- 순서도 알고리즘

    - ex : 은행에서 업무를 보려면 먼저 번호표를 뽑고 기다리다 순서가 되면 업무를 마치고 나오는 형태

### 선택형 구조

- 여러 가지 선택 항목 중 자신의 기준에 가장 적합한 항목을 선택하는 구조

### 반복형 구조

- 주어진 조건이 참일 경우 계속해서 반복하고 거짓이 될 경우 반복을 멈추는 구조

### 연관 규칙

- 명시된 한계에 따라서 빈번하게 나오는 항목값의 조합을 나타내는 공통된 약속을 의미
- 특정 사건이 발생했을 때 빈번하게 발생하는 또 다른 규칙을 발견하는 규칙 기반 학습임

### 패턴 찾기
